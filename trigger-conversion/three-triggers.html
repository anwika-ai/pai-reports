Upgrade triggers — funnel & conversion

presentations.ai · last 7 days (IST) · window 2026-06-18 → 2026-06-25 (12:07 snapshot) · 3 triggers, all measured the same way · payments verified in MySQL is_stripepayments.

How to read this. Each "trigger" is a PRO/GOLD button in the app. When a free user clicks it, the same "Upgrade to Pro" popup appears. From there a person does exactly one of three things — and some later pay:
👆 Clicks a PRO button → sees the popup ✅ Clicks "Upgrade to Pro" ❌ Closes the popup (discards) 🚪 Leaves without acting 💰 Later pays = conversion

A payment only counts if it happens after the click. "Discarded + Clicked + Left" always add up to "Saw the popup" — no one is counted twice.

How the numbers were built & what to trust

  • One window, one method. All three triggers use the identical 7-day window and the same popup events, so they're directly comparable.
  • "Saw the popup" = the upgrade popup was shown after the user clicked that PRO/GOLD button (ClickHouse payment_trigger_show, attributed by the button inside data.source).
  • "Clicked / Discarded / Left" come from payment_trigger_action (clicked Upgrade = success, closed = cancel). Each person is classified once, so the three always sum to "Saw the popup."
  • "Paid" = a real captured payment in MySQL is_stripepayments that landed after the click (joined ClickHouse userId → is_users.id → is_stripepayments.userid). This is more reliable than ClickHouse's own success event, which under-counts.
  • Thin samples: Invite (2 buyers) and locked features (1 buyer) are too small to rank by conversion rate — read them as directional, not precise. Create project (13) is the only one with a usable base.
  • These three are individual buttons; many other buttons also open the same popup, so they don't add up to the site-wide upgrade total.
